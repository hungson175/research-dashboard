1. Vietnam Cyber Threat Surge: 3x Breaches vs 2024

H1 2025 saw 191 data breaches (3x increase), 530K DDoS attacks, and AI-powered phishing targeting mobile wallets. Financial sector absorbed 71% of fraudulent site attacks, with Vietnam ranking 3rd in ASEAN for phishing exposure.

2. Open Banking Phased Rollout: 3-Stage API Compliance

Circular 64/2024 mandates 3-phase API implementation: customer queries (12mo), consent access (18mo), payment initiation (24mo) by March 2027. MoMo must upgrade infrastructure to OAuth 2.0 + TLS 1.2+ standards while maintaining 31M user trust.

4. CBDC Integration: Research Phase with 2026+ Timeline

SBV assigned Viettel/MobiFone to research CBDC (Directive 942), still in early stages with no active pilot. Resolution 5/2025 pilots crypto assets (5-year sandbox), but CBDC rollout likely post-2026, giving MoMo time to prepare API security for eventual integration.

5. Adaptable Security Models: UK, EU, Singapore Frameworks

Global leaders offer proven models: UK's PSD2 mandates 3D Secure 2.2 + TRA exemptions; Singapore's MAS requires 2FA + kill switches + 24/7 monitoring; EU's EBA emphasizes API encryption + fraud reporting. MoMo can adapt these for Vietnam's 44% unbanked population and rural connectivity challenges.

Strategic Recommendations for MoMo

  • Immediate (Q1 2026): Complete Phase 1 Open API migration (OAuth 2.0, TLS 1.2+, developer sandbox) before Sep 2025 deadline; integrate iProov biometrics across all 31M user accounts.
  • Security Infrastructure: Deploy AI-driven fraud detection (real-time monitoring like MAS), implement zero-trust architecture, and establish 24/7 SOC with threat intelligence feeds from Kaspersky/regional CERTs.
  • Compliance Framework: Align with SBV Decision 2345 biometric mandates, adopt PSD2-style Transaction Risk Analysis (TRA) exemptions for low-value payments, and prepare for Decree 94 fintech sandbox participation.
  • User Education: Launch anti-phishing campaigns targeting 44% unbanked/rural users, deploy in-app security training (inspired by Singapore's "Be Safe Online"), and introduce consent dashboards for API data sharing transparency.
  • CBDC Readiness: Monitor SBV research developments, pilot offline payment capabilities (per IMF recommendations), and design CBDC-compatible APIs with quantum-resistant cryptography for future-proofing.
  • Ecosystem Partnerships: Collaborate with VPBank/VietinBank on Open Banking standards, join ASEAN Cyber-CC for regional threat intelligence, and integrate with VietQR/NAPAS for interoperable payment rails.
  • Risk Mitigation: Implement dynamic linking for payment initiation (Phase 3), cap CBDC holdings to prevent disintermediation (per IMF guidance), and maintain incident response plans with <24hr breach notification to SBV.
